# SolPay Server — SEO Marketing Reference

content structure rules for ranking, on-page optimization, and link-building principles.
Every blog skill for SolPay should load this alongside the product-marketing skill.

## Keyword Strategy Overview

SolPay Server sits at the intersection of three keyword universes:

1. **Crypto payment infrastructure** (broad, competitive, high volume)
2. **Solana ecosystem tools** (medium volume, medium competition, highly relevant)
3. **Self-hosted / open-source payment alternatives** (lower volume, lower competition, high purchase intent)

The opportunity is in cluster 3 combined with cluster 2. Competing for "crypto payment
gateway" against Coinbase Commerce and BitPay is a losing bet. Competing for "self-hosted
Solana payments" or "BTCPay Server Solana alternative" is winnable.

## Search Intent Mapping

Always match content to the dominant intent of the target keyword.

| Intent Type | Example Keywords | Content Approach |
|---|---|---|
| **Informational** | "what is self-hosted payments," "how does Solana Pay work" | Educate, build authority, soft CTA at bottom |
| **Commercial / Comparison** | "best Solana payment gateway," "BTCPay vs Solana" | Compare, position, feature table, medium-force CTA |
| **Transactional** | "install SolPay Server," "Solana WooCommerce plugin" | Get them set up fast. Lead with the action, CTA is immediate |
| **Navigational** | "SolPay Server GitHub," "SolPay docs" | Don't target these with blog content — they're already users |

For skyscraper posts, the intent arc is: informational at the top, commercial in the
middle, transactional CTA at the bottom.

## On-Page SEO Rules

These apply to every blog post, landing page, and long-form piece.

### Title Tag
- Primary keyword in the title, ideally near the front
- Under 65 characters (prevents truncation in SERPs)
- No clickbait. Accurately reflect the content.
- Format options: `[Keyword]: [Benefit]` or `How to [Keyword]` or `[Number] Ways to [Keyword]`

### Meta Description
- 140-160 characters
- Primary keyword appears naturally (not stuffed)
- Include a soft CTA or benefit statement
- Does not have to match the intro — it's a sales pitch for the click

### H1
- One H1 per page
- Contains the primary keyword
- Usually the same as or close to the title tag

### H2 and H3 Structure
- H2s break the content into major sections
- H3s go inside H2 sections for subtopics
- Use secondary keywords and LSI terms naturally in headings
- Do not keyword-stuff headings — they should read like useful section labels

### URL Slug
- Lowercase, hyphenated, no stop words
- Primary keyword in slug
- Keep it short: `/blog/solana-payment-gateway` not `/blog/here-is-how-to-set-up-a-solana-payment-gateway-for-your-store`

### Internal Links
- 3-6 internal links per post to contextually related pages or posts
- Anchor text must be descriptive and include target keywords where natural
- Never use "click here" or "read more" as anchor text

### External Links
- Link to credible, authoritative sources: Solana Foundation docs, academic sources,
  major publications, official project pages
- Open in new tab
- 2-4 external links per long-form post is appropriate

### Image Optimization
- Every image has descriptive alt text with a keyword where natural
- Compress images before publishing (WebP preferred)
- File names are descriptive: `solana-payment-gateway-setup.webp` not `image1.png`

### Word Count Guidelines
- Short-form SEO post: 800-1,400 words
- Standard product / how-to post: 1,200-2,000 words
- Skyscraper / comprehensive guide: 2,500-4,500 words
- Do not pad to hit word count. Every sentence should earn its place.

## Content Freshness and E-E-A-T

Google's quality evaluator guidelines (E-E-A-T: Experience, Expertise, Authoritativeness,
Trustworthiness) matter for crypto content, which falls under YMYL (Your Money or Your Life).

**What this means in practice:**
- Include specific, verifiable data and link to sources
- Acknowledge trade-offs honestly (self-hosting has maintenance overhead — say it)
- Don't make performance claims without data backing them
- Mention the open-source nature and link to GitHub — it signals transparency
- Update posts when Solana network stats or product features change

## Keyword Research Process for New Posts

When selecting a keyword for any new piece of content, follow this process:

1. Start with the topic area and generate 10-15 candidate keywords
2. Check estimated search volume (use keyword tools or inference from content competition)
3. Assess competition by looking at what currently ranks:
   - Are the top 10 results major brands or niche sites?
   - Do the top results have high domain authority?
   - Is there a content gap — i.e., the top results answer a different question than the one searchers actually have?
4. Assess intent: does the keyword intent match what SolPay content can satisfy?
5. Check whether SolPay already has a post targeting this keyword (avoid cannibalization)
6. Present the keyword set for approval before writing

**Cannibalization rule:** Two posts should not target the same primary keyword. If a post
already covers a keyword, either update the existing post or find a complementary angle.

## LSI and Semantic Terms to Weave In

These terms signal topical depth to Google without being target keywords themselves.

**For self-hosting content:** private key management, node operator, VPS deployment, open
source, server setup, webhook, API endpoint, invoice generation, payment confirmation

**For Solana content:** Proof of History, Sealevel runtime, SPL token standard, Phantom
wallet, Solana RPC, transaction signature, lamports, Solana validator

**For payment infrastructure content:** payment processor, checkout flow, deposit address,
invoice expiry, settlement, chargeback, fund custody, gateway fee structure

## Link Building Signals Worth Pursuing

For a new project like SolPay Server, organic links come from:
- Being featured in Solana ecosystem directories and newsletters
- GitHub stars and README links from developers who adopt the project
- Community posts on Reddit (r/solana, r/Bitcoin, r/selfhosted) and Hacker News
- Being mentioned in comparison posts on crypto/payments blogs
- Tutorial and integration content that other developers link to

Content that naturally earns links: comparison guides, technical tutorials, data-driven posts,
definitive how-to posts that become the reference for a topic.
